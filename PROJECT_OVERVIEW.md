
# Обзор проекта: Платформа для мероприятий MNU

## 1. Общее представление

**MNU Events** — это веб-платформа, разработанная для централизации и упрощения управления мероприятиями в университетской среде. Она представляет собой единое пространство, где студенты, организаторы и администраторы могут взаимодействовать с событиями, клубами и регистрациями.

Проект решает проблему разрозненности информации о мероприятиях, предоставляя удобный и современный интерфейс для всех участников образовательного процесса.

## 2. Целевая аудитория

Платформа предназначена для трёх основных групп пользователей:

- **Студенты:** Основные пользователи системы. Они могут просматривать список предстоящих мероприятий, фильтровать их по категориям, регистрироваться на участие и отслеживать свои регистрации.
- **Организаторы (Клубы):** Представители студенческих клубов или других организаций. Они могут создавать и управлять мероприятиями, отслеживать количество зарегистрированных участников и управлять посещаемостью.
- **Администраторы:** Пользователи с полными правами доступа. Они управляют всеми аспектами платформы: могут создавать, редактировать и удалять любые мероприятия, управлять пользователями, клубами и системными настройками.

## 3. Ключевые функции

Платформа обладает богатым функционалом, который включает:

- **Аутентификация и авторизация:**
  - Безопасная регистрация и вход в систему с использованием JWT-токенов.
  - Верификация пользователей по электронной почте.
  - Разделение прав доступа на основе ролей (студент, организатор, администратор).

- **Управление мероприятиями:**
  - Создание, редактирование и удаление мероприятий.
  - Возможность добавлять детальную информацию: название, описание, дата, время, место, категория (например, спорт, наука, творчество).
  - Управление статусами мероприятий (например, "предстоящее", "завершено", "отменено").

- **Система регистрации:**
  - Студенты могут регистрироваться на мероприятия в один клик.
  - Если количество мест ограничено, система поддерживает "список ожидания" (waitlist).
  - Пользователи могут просматривать свои активные регистрации в личном кабинете.

- **Управление клубами:**
  - Администраторы могут создавать и управлять студенческими клубами.
  - За каждым клубом закрепляется роль организатора.

- **Отслеживание посещаемости:**
  - Функция "check-in" для подтверждения присутствия участников на мероприятии.

- **Фильтрация и поиск:**
  - Удобная система фильтрации мероприятий по дате, категории и статусу.

## 4. Технический стек

Проект построен на современном и надёжном стеке технологий:

- **Бэкенд:**
  - **Фреймворк:** NestJS (TypeScript)
  - **База данных:** PostgreSQL
  - **ORM:** Prisma
  - **Аутентификация:** JWT (JSON Web Tokens)

- **Фронтенд:**
  - **Библиотека:** React (JavaScript)
  - **Стилизация:** Tailwind CSS
  - **Сборщик:** Vite

- **Инфраструктура:**
  - **Контейнеризация:** Docker

## 5. Цель и ценность проекта

**Цель проекта** — создать централизованную, эффективную и удобную экосистему для организации и участия в студенческой жизни.

**Ценность для пользователей:**

- **Для студентов:** Упрощает поиск интересных событий и участие в них, помогает быть в курсе университетской жизни.
- **Для организаторов:** Предоставляет мощный инструмент для анонсирования и управления мероприятиями, автоматизируя рутинные задачи.
- **Для администрации:** Даёт полный контроль над процессами, связанными с мероприятиями, и обеспечивает порядок в организации студенческой деятельности.

Этот проект необходим для повышения вовлечённости студентов, улучшения коммуникации между клубами и администрацией, а также для создания более яркой и насыщенной студенческой жизни.
